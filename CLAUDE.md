# Project Guide

This file tells your AI engineer how to work on THIS project. It's yours - edit
or delete anything. The defaults below just keep your app clean and readable.

## How to work (Karpathy's rules)
1. Think before coding. State assumptions out loud; if the ask is unclear, ask instead of guessing.
2. Smallest change that works. Touch only what the task needs; match the existing style.
3. Goal-driven. Define "done" as something you can check, then loop until it passes.
4. Keep it simple. Prefer the boring, obvious solution over the clever one.

## UI must-haves (keep it coherent & readable)
- Max 2 font families total (one for text, one for headings) - no more.
- Type sizes from a fixed scale (e.g. 14 / 16 / 20 / 24 / 32) - never random values.
- Body text at least 16px, line-height at least 1.5, lines no wider than ~75 characters.
- Text vs background contrast at least 4.5:1. If it's hard to read, it's wrong.
- Spacing on a 4px scale (4 / 8 / 12 / 16 / 24 / 32) - no arbitrary 7px/13px.
- One consistent look across every screen: same colors, spacing, buttons, motion.
- Works on a phone: check at 390px wide - no sideways scroll, tap targets at least 44px.
- Animations smooth and eased, never abrupt.

## Good habits
- Verify before saying done: run it, click it, watch it actually work.
- Don't invent requirements. If unsure what the user wants, ask one sharp question.
